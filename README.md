# 听书知识库项目需求说明

## 1. 项目背景

面向听书平台场景，建设一个智能听书知识库系统。系统需要接入有声书信息、书籍简介、作者信息、听书笔记、推荐运营资料、用户评论摘要和常见问答等内容，并提供书籍推荐、书籍详情问答、听书场景推荐和知识问答能力。

## 2. 项目目标与功能范围

**内容导入**
- 支持有声书信息、书籍简介、作者介绍、听书笔记、推荐运营资料、用户评论摘要和常见问答等内容导入，导入过程支持任务状态追踪与元数据管理（书名、作者名、内容类型、类别、来源文件名等）

**书籍推荐**
- 可按书名、作者、类别、场景标签等条件查找，返回书籍列表、书籍特色、适合人群和推荐理由

**书籍详情**
- 可查询书籍简介、作者介绍、有声书时长、演播信息、内容标签、听书亮点和常见问题

**内容检索**
- 可定位书籍资料与听书笔记，返回书名、作者名、内容类型、文件名等来源信息

**知识问答**
- 基于知识库内容生成答案，返回必要的引用信息

**交互能力**
- 支持单轮问答、多轮对话、流式输出和历史记录管理

**质量要求**
- 系统应具备基础异常处理能力，避免导入失败导致服务整体不可用；查询结果应尽量准确、可解释、可追溯，流式输出应具备较好的实时性；各模块应具备清晰分层，便于后续扩展

## 3. 角色说明

### 3.1 主要使用对象

- 听众：查询书籍介绍、作者信息、听书笔记、推荐理由和常见问题
- 内容运营：辅助整理书籍资料、生成推荐内容、回答常见问题、快速检索知识点

### 3.2 协同角色

- 内容编辑：管理书籍资料、上传有声内容、校验知识库内容

## 4. 典型业务场景

### 4.1 书籍推荐场景

当用户提出”有哪些科幻类有声书””推荐一个适合通勤听的悬疑小说”之类的问题时，系统应根据书籍简介、有声书信息、推荐语、评论摘要等资料返回书籍列表、书籍特色、适合人群和推荐理由等内容。

### 4.2 书籍详情场景

当用户指定某一书籍时，系统应能够返回书籍简介、作者介绍、有声书时长、演播信息、内容标签、听书亮点和常见问题等详细信息。

### 4.3 内容检索场景

当用户询问某本书讲什么、适合谁听、有哪些核心看点或某个主题的解释时，系统应优先返回相关书籍简介、听书笔记、常见问答和评论摘要，并在需要时生成结构化回答。返回结果中应尽量包含书名、作者名、内容类型、文件名等来源信息。

### 4.4 听书笔记场景

当用户提出”《三体》有哪些精彩书评”或”查询红楼梦相关的听书笔记”等问题时，系统应能定位到对应书籍或生成笔记推荐，并返回笔记内容、评论摘要、推荐理由等。返回结果中应包含书名、作者名、内容类型等来源信息。

## 5. 内容要求

知识库中的每条内容建议包含以下字段：

- 内容正文
- 内容类型：有声书信息 / 书籍简介 / 作者介绍 / 听书笔记 / 推荐运营资料 / 用户评论摘要 / 常见问答
- 书名
- 作者名
- 条目名称
- 类别/标签（小说/非小说/儿童/教育等）
- 有声书时长（有声书类型内容）
- 来源文件名
- 来源路径或资源链接

## 6. 后续扩展方向

- 增加书籍后台管理与知识库管理页面
- 增加文档增量更新能力
- 增加权限控制与多租户支持
- 增加个性化推荐、听书路径等听书能力
- 增加有声书语音检索能力，支持基于语音转文本、字幕或时间轴文本定位相关片段
- 有声书检索结果可返回书名、作者、开始时间、结束时间和片段摘要等信息
- 增加多模态检索，支持封面图片与内容联合检索
